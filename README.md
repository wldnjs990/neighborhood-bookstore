# 📗 동네책방 서비스 개인 리팩토링 & 기능 개발용 레포지토리
- 본 프로젝트는 싸피 관통 프로젝트 "동네책방" 프로젝트를 개인적으로 추가 개발하기 위해 만들어졌습니다.
- 프로젝트 원본 주소 : [주소창]

## 🥅 프로젝트 목표
- 프로젝트 목표는 크게 3가지입니다.
  1. **프론트 프레임워크 변경(Next.js)**
  2. **실시간 채팅 기능 구현**
  3. **로컬 DB 전환(SQLite -> ?) 및 배포**

### 📌 1. 프론트 프레임워크 변경(Next.js)
- Vue.js로 개발된 SPA 프로젝트에서 React + Next.js를 사용한 빌드타임 렌더링 프로젝트로 마이그레이션
-**왜 하는가?**
  - 빠른 개발 및 배포를 중점으로 사용된 Vue 프로젝트에서, 본격적인 고도화를 진행하기 위해 typeScript와 React를 함께 사용해 서비스 안정성을 개선하기 위함
  - Next.js를 도입함으로 정적인 페이지를 빌드타임에 렌더링하는 전략을 사용해 사용자의 TTI 경험을 개선하기 위함

### 📌 2. 실시간 채팅 기능 구현
- WebSoket 통신을 도입해 실시간 채팅기능 구현
- **왜 하는가?**
  - 동네책방의 **"중고책 거래"** 기능을 더욱 고도화 하기 위함
  - 현재 중고책 거래 채팅 기능은 카카오톡 오픈 채팅방 URL을 연결해 서드파티로 사용중임
  - 사용자가 직접 서드파티앱에서 채팅을 만들어야하고, 서비스가 타 서비스에 의존하고있는 문제점을 해결하기 위함

### 📌 3. 로컬 DB 전환(SQLite -> ?) 및 배포
- SQLite 로컬 DB에서 클라우드 DB(MySQL, MongoDB 등)로 DB 변경 및 프로젝트 배포
- **왜 하는가?**
  - 서비스 배포에 있어 로컬 DB로는 배포가 불가능함
  - 배포가 가능한 클라우드 DB를 사용해 프로젝트 배포까지 진행 예정
